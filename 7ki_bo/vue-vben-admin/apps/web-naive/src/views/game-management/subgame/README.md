# 子游戏管理 - 文件导入功能

## 功能概述

子游戏管理模块提供了完整的游戏导入功能，支持从Excel和CSV文件批量导入游戏数据到数据库。

## 主要特性

### 1. 文件导入流程

- **第一步：文件上传** - 选择平台、币种，上传Excel/CSV文件
- **第二步：数据预览** - 验证数据格式，显示错误信息
- **第三步：导入处理** - 显示实时进度，处理数据导入
- **第四步：结果展示** - 显示导入结果统计和详情

### 2. 文件格式支持

- **Excel格式**: .xlsx, .xls
- **CSV格式**: .csv
- **模板下载**: 提供标准模板文件

### 3. 数据验证

- **必填字段验证**: 游戏名称(中文)、游戏名称(英文)、游戏类型、游戏ID
- **格式验证**: 游戏ID格式检查（字母、数字、下划线、横线）
- **重复性检查**: 防止重复导入相同游戏

## 文件格式要求

### 标准列格式

根据提供的截图，标准文件格式包含以下列：

| 列名             | 类型 | 必填 | 说明         |
| ---------------- | ---- | ---- | ------------ |
| NO.              | 数字 | 否   | 序号         |
| Game Name (中文) | 文本 | 是   | 游戏中文名称 |
| Game Name (英文) | 文本 | 是   | 游戏英文名称 |
| 游戏类型         | 文本 | 是   | 游戏分类     |
| GameID           | 文本 | 是   | 游戏唯一标识 |
| Icon             | URL  | 否   | 游戏图标链接 |

### 数据示例

```csv
NO.,Game Name (中文),Game Name (英文),游戏类型,GameID,Icon
1,红飞机,Aviator,桌面游戏,spribe_01,https://example.com/icon1.png
2,黑杰克,Blackjack,桌面游戏,evolution_bj,https://example.com/icon2.png
3,轮盘,Roulette,桌面游戏,evolution_roulette,https://example.com/icon3.png
```

## 技术实现

### 客户端文件解析

- 使用原生JavaScript解析CSV文件
- 支持带引号的字段和特殊字符
- 实时数据验证和错误提示

### 组件架构

```
GameImportDialog.vue          # 主导入对话框
├── 上传步骤                  # 文件选择和基本配置
├── 预览步骤                  # 数据验证和预览
├── 导入步骤                  # 实时进度显示
└── 结果步骤                  # 导入结果展示

fileParser.ts                 # 文件解析工具
├── parseCSV()                # CSV解析
├── validateGameData()        # 数据验证
├── downloadTemplate()        # 模板下载
└── 其他工具函数
```

### 数据流程

1. **文件上传**: 用户选择文件和配置
2. **客户端解析**: 解析文件内容为结构化数据
3. **数据验证**: 验证必填字段和格式
4. **预览确认**: 显示解析结果和错误信息
5. **批量导入**: 调用API保存到数据库
6. **结果反馈**: 显示成功/失败统计

## 使用方法

### 1. 准备数据文件

- 下载提供的模板文件
- 按照格式要求填写游戏数据
- 确保必填字段完整

### 2. 导入操作

1. 点击"导入游戏"按钮
2. 选择所属平台和默认币种
3. 上传准备好的数据文件
4. 点击"解析文件"进行验证
5. 检查预览数据和错误信息
6. 点击"开始导入"执行导入
7. 查看导入结果和统计

### 3. 错误处理

- 查看错误信息，修正数据文件
- 重新上传修正后的文件
- 系统会跳过已存在的游戏

## 注意事项

1. **文件大小限制**: 建议单次导入不超过1000条记录
2. **数据格式**: 严格按照模板格式准备数据
3. **游戏ID唯一性**: 确保GameID在平台内唯一
4. **网络稳定**: 导入过程中保持网络连接稳定
5. **备份数据**: 导入前建议备份现有数据

## API接口

### 导入相关接口

- `POST /games/import-from-file` - 批量导入游戏
- `GET /games/import-template` - 下载导入模板
- `POST /games/parse-import-file` - 解析导入文件（可选）

### 数据结构

详见 `#/api/game/subgame.ts` 中的类型定义：

- `ImportGameData` - 导入游戏数据结构
- `FileImportParams` - 导入请求参数
- `FileImportResponse` - 导入响应结果
